To create a MySQL database that is UTF-8 compliant 
from the command line, you can follow these steps:

### Step 1: Open Terminal
Open the Terminal application on your MacBook.

### Step 2: Log in to MySQL
Log in to your MySQL server using the `mysql` command. 
Replace `your_username` with your MySQL username and 
`your_password` with your MySQL password.

```sh
mysql -u your_username -p
```

You will be prompted to enter your MySQL password.

### Step 3: Create the Database
Once logged in, you can create a new database with 
UTF-8 encoding using the following SQL command. Replace 
`your_database_name` with the desired name for your database.

```sql
CREATE DATABASE your_database_name 
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Here, `utf8mb4` is the recommended character set for full 
UTF-8 support, and `utf8mb4_unicode_ci` is the collation 
that ensures proper sorting and comparison of Unicode characters.

### Example

Here's a complete example of creating a UTF-8 compliant 
database named `my_utf8_database`:

1. Open Terminal.
2. Log in to MySQL:

```sh
mysql -u your_username -p
```

3. Create the database:

```sql
CREATE DATABASE my_utf8_database 
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

4. Verify the database creation:

```sql
SHOW DATABASES;
```

5. Optionally, check the character set and collation 
of the newly created database:

```sql
USE my_utf8_database;
SHOW VARIABLES LIKE 'character_set_database';
SHOW VARIABLES LIKE 'collation_database';
```

### Summary

By following these steps, you will have created a MySQL database 
that is fully UTF-8 compliant, ensuring proper handling and 
storage of Unicode characters.
