# Slide 2 — Why This Matters
Modern organizations generate:
- Structured data (ERP, CRM)
- Semi-structured data (JSON, logs)
- Unstructured data (images, PDFs)
- Streaming data (IoT, clickstreams)

Traditional warehouses struggle to scale economically.

---

# Slide 3 — What Is a Data Lake?
A **Data Lake** is a centralized repository that stores raw data in its native format.

Key traits:
- Schema-on-read
- Cheap storage (object storage)
- Scalable
- Flexible

---

# Slide 4 — Data Lake Core Principles
- Store everything
- Transform later
- Keep historical detail
- Decouple storage from compute

---

# Slide 5 — Data Lake Architecture (Conceptual Diagram)

   ↓
Object Storage (S3 / HDFS / ADLS)
   ↓
Query Engines (DuckDB / Spark / Trino)
   ↓
Analytics / BI / ML

# Slide 6 — Data Lake vs Data Warehouse

| Feature | Data Warehouse | Data Lake |
|----------|---------------|------------|
| Schema | Schema-on-write | Schema-on-read |
| Data Types | Structured | All formats |
| Cost | High | Low |
| Flexibility | Moderate | High |

# Slide 18 — Case Study: Flights Dataset

Data:
- 6.7M rows
- Parquet format
- 2022 domestic flights

Discovery process:
1. Inspect schema
2. Cast types
3. Build bronze/silver/gold

---

# Slide 19 — Flights Bronze Layer

CREATE VIEW flight_raw AS
SELECT * FROM read_parquet('flight_parquet/*.parquet');

---

# Slide 20 — Flights Silver Layer

CREATE VIEW flight_silver AS
SELECT
    CAST(Year AS INT) AS year,
    CAST(Month AS INT) AS month
FROM flight_raw;

---

# Slide 21 — Flights Gold Layer

CREATE VIEW flight_gold AS
SELECT
    *,
    CASE WHEN dep_delay_min > 15 THEN 1 ELSE 0 END AS dep_delayed_15
FROM flight_silver;
