# Data Lake Metadata Discovery & Governance

## Executive-Level Lecture Deck (35 Slides)

### For Data Warehousing & Lakehouse Courses

------------------------------------------------------------------------

# Slide 1 --- Executive Context

Modern data platforms are no longer just warehouses.

They are ecosystems: 

- Raw files 
- Streaming feeds 
- External data 
- Multiple formats 
- Distributed storage

**Metadata is what makes this ecosystem usable.**

------------------------------------------------------------------------

# Slide 2 --- The Core Problem

**A data lake without metadata is:**

-   A dumping ground
-   Undiscoverable
-   Untrustworthy
-   Unmanageable

**Metadata enables:**

- Discovery 
- Governance 
- Optimization
- Trust

------------------------------------------------------------------------

# Slide 3 --- What is Metadata?

Metadata = "Data about data"

Three main categories:

1.  Technical Metadata

2.  Business Metadata

3.  Operational Metadata

------------------------------------------------------------------------

# Slide 4 --- Technical Metadata

Includes:

-   Schema
-   Column types
-   File format
-   Partition structure
-   Storage location
-   Statistics

Example (DuckDB):

``` sql
DESCRIBE 
SELECT * 
FROM read_parquet('data/*.parquet');
```

Example: 


# Slide 5 --- Business Metadata

Includes:

-   Column definitions
-   Data ownership
-   KPI definitions
-   Data classification

Example: 

- What does "`DepDelayMinutes`" actually mean? 
- Is it minutes or hours?

------------------------------------------------------------------------

# Slide 6 --- Operational Metadata

Includes:

-   File size
-   Row count
-   Last modified date
-   Data freshness
-   Pipeline lineage

Example:

``` sql
SELECT file_name, file_size 
FROM parquet_metadata('data/*.parquet');
```

``` sql
D SELECT
      file_name,
      SUM(total_compressed_size) AS compressed_size_bytes,
      SUM(total_uncompressed_size) AS uncompressed_size_bytes
  FROM parquet_metadata('./chap07/users.parquet')
  GROUP BY file_name;
```

# Slide 7 --- Data Lake Discovery Challenge

### In traditional DW:

-   Schema defined first
-   Controlled ingestion
-   Known structure

### In Data Lake:

-   Files arrive continuously
-   Multiple producers
-   Schema may evolve

------------------------------------------------------------------------

        /carriers

- Min/Max values

# Slide 12 --- Column-Level Statistics

``` sql
SELECT column_name, 
       min_value, 
       max_value
FROM parquet_metadata('data/*.parquet');
```

This helps with: - Partition pruning - Predicate pushdown
